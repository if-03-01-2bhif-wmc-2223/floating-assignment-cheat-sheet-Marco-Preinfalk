How are resources (html pages, images, etc) addressed in the web?
- The target of an HTTP request is called a “resource”, whose nature isn’t defined further; it can be a document, a photo, or anything else. Each resource is identified by a Uniform Resource Identifier (URI) used throughout HTTP for identifying resources.

Web Server

What is a web server?
- A web server is computer software and underlying hardware that accepts requests via HTTP (the network protocol created to distribute web content) or its secure variant HTTPS.
What does a web server?
- A user agent, commonly a web browser or web crawler, initiates communication by making a request for a web page or other resource using HTTP, and the server responds with the content of that resource or an error message. A web server can also accept and store resources sent from the user agent if configured to do so.
How can content be stored on a web server?
- There are two main web storage types: local storage and session storage, behaving similarly to persistent cookies and session cookies respectively.
How is content in webpages described?
-The web browser gets the data and renders it so that the user can view the website

Web Client

What is a web client?
- A web browser (also referred to as an Internet browser or simply a browser) is application software for accessing the World Wide Web or a local website.
What does a web client?
- When a user requests a web page from a particular website, the web browser retrieves its files from a web server and then graphically renders the page on the user’s screen.

Graphic Formats

JPEG:
- Am besten geeignet für Fotos und Bilder mit mehreren Farben
- Verwendet verlustbehaftete Komprimierung, sodass ein Teil der Bildqualität verloren geht, wenn das Bild gespeichert wird
- unterstützt keine Animationen
GIF:
- Am besten geeignet für einfache Grafiken und Bilder mit einer begrenzten Farbpalette
- Verwendet verlustfreie Komprimierung, um die Bildqualität beim Speichern von Bildern zu erhalten
- Unterstützt Animationen
PNG:
- Am besten geeignet für transparente Bilder, Grafiken mit scharfen Kanten und Bilder, die eine hohe Auflösung erfordern
- Verwendet verlustfreie Komprimierung, um die Bildqualität beim Speichern von Bildern zu erhalten
- unterstützt keine Animationen
ICO:
- Am besten für kleine Icons und Favicons
- Verwendet verlustfreie Komprimierung, um die Bildqualität beim Speichern von Bildern zu erhalten
- unterstützt keine Animationen
SVG:
- Am besten geeignet für skalierbare Vektorgrafiken (Grafiken, die ohne Qualitätsverlust hochskaliert werden können).
- Es wird keine Komprimierung verwendet, daher kann die Dateigröße größer sein als bei anderen Formaten unterstützt keine Animationen
MPEG / MP4:
- Am besten für bewegte Bilder und Videos
- Verwendet verlustbehaftete Komprimierung, sodass ein Teil der Videoqualität beim Speichern des Videos verloren geht
- unterstützt Animationen
